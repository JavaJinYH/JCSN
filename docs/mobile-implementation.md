# 折柳建材 - 移动端实施文档

## 一、项目概述

### 1.1 目标

为折柳建材管理系统开发移动端功能，支持：

- **查看数据**：库存、订单、客户（只读）
- **单据处理**：待拍照单据列表、拍照上传关联单据
- **数据同步**：通过 Syncthing 在局域网同步数据

### 1.2 目标用户

| 用户 | 设备 | 场景 |
|------|------|------|
| 爸爸/妈妈 | 各持有一台手机/平板 | 查看库存、订单，拍照上传签字单 |

**注意**：送货员不接入系统，签字单照片通过微信/电话发给老板，由老板上传到系统。

### 1.3 多设备场景

**家中两台手机同时使用**，需支持：

```
┌─────────────────┐        WiFi         ┌─────────────────┐
│   爸爸的手机      │ ◄──────────────►  │                   │
│  - 查看库存/订单  │                    │                   │
│  - 拍照上传     │                    │   店里电脑（主）   │
├─────────────────┤                    │  - SQLite 数据库  │
│   妈妈的手机      │ ◄──────────────►  │  - Photos目录    │
│  - 查看库存/订单  │    局域网API      │  - Syncthing     │
│  - 拍照上传     │                    │                   │
└─────────────────┘                    └─────────────────┘
```

**关键点**：
1. **仅家庭成员使用** - 不对外开放，保证数据安全
2. **多台手机可同时使用** - 均通过局域网 API 上传照片到主电脑
3. **Syncthing 同步到所有设备** - 主电脑、两台手机都能同步数据
4. **照片上传统一到主电脑** - 所有照片存在主电脑的 Photos 目录
5. **待处理单据实时更新** - 一台手机上传后，其他设备刷新可见

### 1.4 架构方案

```
┌─────────────────┐        WiFi         ┌─────────────────┐
│   手机（移动端）  │ ◄──────────────►  │   店里电脑（主）  │
│  - 移动端UI      │   局域网API上传    │  - SQLite 数据库 │
│  - Syncthing    │                    │  - Syncthing同步 │
│  - Photos目录   │                    │  - Photos目录   │
└─────────────────┘                    └─────────────────┘
```

## 三、技术方案

### 3.1 技术栈

| 部分 | 技术 |
|------|------|
| 移动端 UI | React 18 + TypeScript + TailwindCSS |
| 打包 | Electron + electron-builder（APK） |
| 数据同步 | Syncthing（局域网） |
| 局域网服务 | Express + Node.js（集成到 Electron） |
| 相册/摄像头 | 标准 Web API（无需手机型号适配） |

### 3.2 代码架构：共享核心 + 分离外壳

```
src/
├── services/          # 共享，业务逻辑不变
│   ├── SaleService.ts
│   ├── InventoryService.ts
│   └── ...
├── components/
│   ├── layout/
│   │   ├── AppLayout.tsx        # 桌面端（侧边栏）
│   │   └── MobileLayout.tsx     # 移动端（底部导航）
│   └── ...
└── pages/             # 同一套页面，通过 CSS 响应式适配
    ├── Sales.tsx       # 同一文件
    └── ...
```

**原则**：
1. **Services 层 100% 共享** - 业务逻辑只有一份
2. **页面组件同一份** - 通过 CSS class 响应式适配
3. **一处修改，两端生效** - 改 SalesService.ts，桌面和移动端都用新的
4. **Schema 只有一份** - 不存在多端同步问题

### 3.3 相册/摄像头调用

使用**标准 Web API**，无需针对不同手机型号开发：

| 功能 | 实现方式 |
|------|----------|
| 选择相册照片 | `<input type="file" accept="image/*">` |
| 调用摄像头拍照 | `navigator.mediaDevices.getUserMedia()` |

- ✅ 所有现代浏览器（Chrome、Safari、Edge）都支持
- ✅ Electron 打包的 APK 内部用 WebView，同样支持标准 API
- ✅ 不需要适配华为、小米、OPPO 等不同机型

#### 3.4.1 响应式布局

**方案**：根据屏幕宽度自动切换布局

```javascript
// 判断是否移动端
const isMobile = window.innerWidth < 768;

// 布局切换
if (isMobile) {
  // 底部导航
} else {
  // 侧边栏
}
```

#### 3.4.2 局域网 API 服务

在主电脑上运行一个简单的 Express 服务，支持**多台手机同时访问**：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/pending-documents` | GET | 获取待拍照的单据列表 |
| `/api/document/:id` | GET | 获取单据详情 |
| `/api/document/:id/upload-photo` | POST | 上传照片并关联单据 |
| `/api/inventory` | GET | 查询库存（移动端查看用） |
| `/api/sales` | GET | 查询销售单列表 |
| `/api/purchases` | GET | 查询进货单列表 |

**多设备支持**：
- 服务监听 `0.0.0.0:3456`，局域网内所有设备都可访问
- 无需登录验证（同一WiFi内网，安全）
- 并发上传支持，照片自动关联到对应单据
- 主电脑设置页面显示当前局域网 IP（如 `http://192.168.1.100:3456`）

#### 3.4.3 移动端只读模式

移动端所有编辑操作禁用，只显示：

| 页面 | 移动端模式 |
|------|------------|
| Dashboard | 简化版，只显示关键指标 |
| Inventory | 列表查看，无编辑/新增 |
| Sales | 列表查看，无编辑/新增 |
| Purchases | 列表查看，无编辑/新增 |
| Contacts | 列表查看，无编辑/新增 |
| **MobilePending** | **单据列表 + 拍照上传** |

## 四、详细实施步骤

### 阶段一：环境准备（0.5 天）

1. 检查现有项目依赖
2. 安装必要的移动适配包（react-device-detect）
3. 创建 `electron-builder.json` 配置文件

### 阶段二：响应式布局改造（3-5 天）

#### 任务 2.1：创建移动端布局组件

- 创建 `MobileBottomNav.tsx`（底部导航）
- 创建 `MobileAppLayout.tsx`（移动端总布局）
- 改造 `AppLayout.tsx` 根据屏幕宽度切换布局

#### 任务 2.2：适配关键页面

1. **Dashboard** - 简化为卡片式布局
2. **Inventory** - 调整列表为移动端友好
3. **Sales** - 调整列表为移动端友好
4. **Purchases** - 调整列表为移动端友好
5. **Contacts** - 调整列表为移动端友好

### 阶段三：单据拍照功能（2-3 天）

#### 任务 3.1：创建待拍照单据页面

- 页面：`/mobile/pending-documents`
- 显示销售单和进货单（有照片关联需求的）
- 筛选条件：日期、状态

#### 任务 3.2：单据详情 + 拍照页面

- 查看单据详情（客户、商品、金额）
- 拍照功能（调用摄像头）
- 相册选择功能
- 上传照片，关联到单据

### 阶段四：局域网 API 服务（1-2 天）

#### 任务 4.1：创建 Express 服务

- 位置：`/server/index.js`
- 监听：`0.0.0.0:3456`
- 启用 CORS

#### 任务 4.2：实现 API 接口

1. GET `/api/pending-documents` - 查询待拍照单据
2. GET `/api/document/:id` - 查询单据详情
3. POST `/api/document/:id/upload-photo` - 上传照片

#### 任务 4.3：集成到 Electron

- Electron 启动时自动启动本地服务
- 在设置页面显示当前局域网 IP

### 阶段五：Electron 打包（1 天）

#### 任务 5.1：配置 electron-builder

- 创建 `electron-builder.json`
- 配置 APK 打包选项
- 配置应用签名

#### 任务 5.2：调整 Electron 配置

- 移除移动端不适合的功能（全局快捷键、窗口置顶等）
- 调整窗口默认尺寸（移动端不需要大尺寸）
- 检测并适配移动端屏幕

### 阶段六：Syncthing 配置指导（0.5 天）

- 编写 Syncthing 配置文档
- 包含：安装、同步文件夹设置、权限配置
- 推荐的同步策略（只同步数据库和 Photos 目录）

### 阶段七：测试与优化（1-2 天）

- 安卓真机测试
- 局域网上传测试
- Syncthing 同步测试
- 性能优化

## 六、风险与注意事项

### 6.1 技术风险

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| Electron 打包 APK 失败 | ⭐⭐⭐⭐ | 备用方案：使用 Capacitor 或 Tauri |
| Syncthing 在部分手机上无法运行 | ⭐⭐ | 备用方案：使用其他同步工具或 FTP |
| 局域网连接不稳定 | ⭐⭐ | 支持照片本地缓存，网络恢复后重试 |

### 6.2 业务注意事项

1. **照片关联**：必须确保照片与单据正确关联，避免错配
2. **数据安全**：Syncthing 同步文件夹要设置权限，防止误删
3. **用户体验**：移动端操作要简单直观，减少步骤
