## 一、项目概述

### 1.1 目标

为折柳建材管理系统添加移动端适配，采用**混合架构**：

- **电脑端**：保持现有功能不变（继续用 Electron）
- **手机端**：Capacitor 打包，只读缓存（不修改数据），断网也能看旧数据
- **数据同步**：通过 Syncthing 在局域网同步照片目录（数据库不用同步！）

### 1.2 目标用户

| 用户    | 设备         | 场景              |
| ----- | ---------- | --------------- |
| 爸爸/妈妈 | 各持有一台手机/平板 | 查看库存、订单，拍照上传签字单 |

**注意**：送货员不接入系统，签字单照片通过微信/电话发给老板，由老板上传到系统。

### 1.3 核心原则

1. **同一套代码** - 电脑端和手机端共用 95% 以上代码
2. **不影响电脑端** - 继续优化和调试电脑端，互不影响
3. **渐进式开发** - 先做响应式，再加新功能
4. **离线查看旧数据** - 手机端有只读缓存，断网也能看旧数据
5. **不冲突原则** - 手机端不修改数据，完全避免同步冲突

### 1.4 多设备场景

**家里两台手机 + 店里一台电脑**，数据实时同步：

```
┌─────────────────┐        WiFi         ┌─────────────────┐
│  爸爸的手机（只读缓存）│ ◄────────────► │  店里电脑（唯一权威）│
│  - 在局域网时：看最新数据 │                    │  - SQLite 数据库     │
│  - 不在局域网时：看旧数据 │                    │  - Photos 目录      │
│  - 拍照本地缓存（等网了再上传 │                    │  - Syncthing        │
├─────────────────┤                    │  - 局域网 API 服务  │
│  妈妈的手机（只读缓存）│ ◄────────────► │                     │
│  - 在局域网时：看最新数据 │                    │                     │
│  - 不在局域网时：看旧数据 │                    │                     │
│  - 拍照本地缓存（等网了再上传 │                    │                     │
└─────────────────┘                    └─────────────────┘
         │
         ▼
   Syncthing 同步
         │
         └─→ 只同步 Photos 目录（数据库不用同步！）
```

**关键点**：

1. **仅家庭成员使用** - 不对外开放，保证数据安全
2. **多台手机可同时使用** - 均通过局域网 API 上传照片到主电脑
3. **店里电脑是唯一权威** - 手机端只读，不修改数据，完全避免冲突
4. **照片上传统一到主电脑** - 所有照片存在主电脑的 Photos 目录
5. **待处理单据实时更新** - 一台手机上传后，其他设备刷新可见
6. **离线查看旧数据** - 手机不在局域网也能看最后一次缓存的旧数据，提示"数据可能不是最新"

## 四、已实现功能说明

### 4.1 响应式布局

- 窗口宽度 < 768px：自动显示底部导航栏
- 窗口宽度 ≥ 768px：保持原侧边栏布局
- 电脑端窗口最小宽度 320px（支持缩到手机尺寸）

### 4.2 表格/卡片双布局

- **SaleTable**：电脑端表格，手机端卡片式，包含关键信息
- **Inventory**：电脑端表格，手机端卡片式，包含商品关键信息
- **Purchases**：原本就是卡片式，完美支持
- **Collections**：电脑端表格，手机端卡片式，包含催账完整信息
- **Settlements**：电脑端表格，手机端卡片式，包含主体和欠款信息

### 4.3 响应式分页控件

- **电脑端**：完整分页（首页、上一页、页码、下一页、末页）
- **手机端**：简化分页（首页、上一页、当前/总页数、下一页、末页）

### 4.4 局域网 API 服务

- 运行端口：3456（集成在 Electron 主进程）
- 访问地址：`http://电脑IP:3456`
- 电脑端 Header 实时显示当前 API 地址
- 支持 CORS，移动端浏览器可直接访问

### 4.5 移动端页面

| 页面路径                        | 功能          | 限制        |
| --------------------------- | ----------- | --------- |
| `/mobile/pending-documents` | 待拍照单据列表     | 只能查看      |
| `/mobile/document/:id`      | 单据详情 + 拍照上传 | 只能查看和拍照上传 |
| `/mobile/collections`       | 催账记录列表      | 只读        |
| `/mobile/settlements`       | 挂账主体列表      | 只读        |
| `/mobile/settlements/:id`   | 主体详情 + 订单   | 只读        |

### 4.6 离线缓存机制

- 联网时：自动获取最新数据并缓存到 LocalStorage
- 离线时：显示缓存数据，提示"📴 离线模式"或"💾 缓存数据"
- API 地址保存到 LocalStorage，下次自动填充

## 七、常见问题解答

### Q1：会不会影响我现在优化电脑端？

**完全不会！**：电脑端功能完全不变，你可以继续优化调试，互不影响。

### Q2：是不是要开发一套新系统？

**不是！** 95% 的代码继续用，只是加了响应式和移动端页面。

### Q3：两台手机能不能同时用？

**可以！** 主电脑开启局域网服务，两台手机都能访问。

### Q4：数据怎么同步？

用 Syncthing 只同步 Photos 目录（数据库不用同步，避免冲突！）。

### Q5：手机不在局域网能看数据吗？

**可以！** 显示最后一次在局域网时缓存的旧数据，同时提示"💾 缓存数据"或"📴 离线模式"。

### Q6：手机不在局域网能拍照上传吗？

**可以拍，但需要联网才能上传。** 照片暂时保存在手机本地，等回到局域网后刷新页面上传。

### Q7：店里电脑修改了订单，手机回来会有冲突吗？

**完全不会！** 手机端只读不写，每次在局域网都是直接调用 API 看最新数据，完全没问题。

### Q8：两个人都出门了，都拍了照回来上传，会冲突吗？

**不会！** 照片按时间顺序上传到电脑端，完全没问题。

### Q9：怎么在手机上访问移动端页面？

1. 确保手机和电脑在同一 WiFi 网络
2. 电脑端运行 Electron 应用（API 服务自动启动）
3. 打开电脑端 Header 查看当前 IP 地址（绿色显示）
4. 手机浏览器访问 `http://那个IP:3456`
5. 点击右上角切换到移动端视图

### Q10：IP 地址变了怎么办？

**不用改！** 每次电脑 IP 变化后，电脑端 Header 会自动显示新的 IP，手机端刷新页面输入新地址即可。

| `refactor(mobile):`  | ♻️ 移动端重构      |
| `style(mobile):`     | 🎨 移动端样式      |
| `docs(mobile):`      | 📚 移动端文档      |
| `feat(desktop):`     | 💻 电脑端新功能     |
| `fix(desktop):`      | 🐛 电脑端 Bug 修复 |
| `refactor(desktop):` | ♻️ 电脑端重构      |
| `docs(desktop):`     | 📚 电脑端文档      |

**示例提交：**

```bash
# 移动端提交
git add docs/mobile-implementation.md
git commit -m "docs(mobile): 更新移动端实施进度"

# 电脑端提交
git commit -m "feat(desktop): 添加新的销售报表功能"
```

这样以后回滚代码时，就不会分不清哪些是电脑端改动，哪些是移动端改动了！
