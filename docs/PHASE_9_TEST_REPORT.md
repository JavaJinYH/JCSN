## 3. 警告项分析

### 3.1 需要数据支撑的功能
以下功能需要先有业务数据才能验证，自动化测试无法直接验证：
- 新增进货记录（需要先有进货数据）
- 进货退货功能（需要先有进货记录）
- 销售退货功能（需要先有销售记录）
- 还款功能（需要先有应收账款）
- 添加催账记录（需要先有催账场景）

### 3.2 需要展开/滚动的UI元素
以下元素的定位器在自动化测试中未能精确匹配，但功能本身存在：
- 库存分类筛选
- 商品编辑按钮
- 赊账选项
- 项目详情按钮

### 3.3 特定Tab/弹窗内的功能
以下功能需要进入特定Tab或弹窗后才能验证：
- 配送费配置
- 结算调整
- 销售日报
- 利润统计
- 店铺信息设置
