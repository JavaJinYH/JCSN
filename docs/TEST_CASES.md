# 折柳建材店管理系统 - 自动化测试用例文档

**文档版本**: v1.0
**创建日期**: 2026-04-18
**最后更新**: 2026-04-18
**测试框架**: Playwright
**适用版本**: v4.7+

---

## 1. 测试概述

### 1.1 测试目标

本测试用例文档旨在为折柳建材店管理系统建立完整的自动化测试体系，确保：
- **功能完整性**：所有业务模块功能正常
- **数据一致性**：数据库操作正确，数据关系完整
- **业务流程顺畅**：全流程业务操作可正常执行
- **异常处理健壮**：错误输入和边界条件有合理处理

### 1.2 测试范围

| 测试类型 | 覆盖模块 |
|---------|---------|
| 单功能测试 | 库存管理、进货管理、销售管理、商品管理、客户管理、联系人管理、结账主体管理、项目管理、回扣管理、配送管理、挂账结算、催账记录、对账单、照片管理、报表统计、系统设置 |
| 全流程测试 | 从项目创建到年终结算的完整业务周期 |
| 异常测试 | 边界条件、错误输入、并发操作 |

### 1.3 测试用例编号规则

```
TC-[模块缩写]-[序号]
模块缩写：
  INV - 库存管理 (Inventory)
  PUR - 进货管理 (Purchases)
  SAL - 销售管理 (Sales)
  PRD - 商品管理 (Products)
  CST - 客户管理 (Customers)
  CNT - 联系人管理 (Contacts)
  ENT - 结账主体管理 (Entities)
  PRJ - 项目管理 (Projects)
  RBT - 回扣管理 (Rebates)
  DLV - 配送管理 (Deliveries)
  STT - 挂账结算 (Settlements)
  COL - 催账记录 (Collections)
  STM - 对账单 (Statements)
  PHT - 照片管理 (PhotoManagement)
  RPT - 报表统计 (Reports)
  SET - 系统设置 (Settings)
  E2E - 端到端测试
  ERR - 异常处理测试
```
