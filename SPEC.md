# 建材水暖店管理系统 - 项目规格说明书

## 1. 项目概述

### 1.1 项目名称
折柳建材店管理系统 (ZheLiu Building Materials Management System)

### 1.2 项目类型
桌面应用程序 (Electron-based Desktop Application)

### 1.3 核心功能概述
为夫妻经营的建材水暖小店提供库存管理、销售管理、基础数据管理和报表统计的全面解决方案，解决手工记账导致的库存混乱、账目不清、销售管理困难等问题。

### 1.4 目标用户
- 店主（夫妻）：日常经营管理者
- 特点：非专业计算机用户，需要简洁易用的界面

---

## 2. 设计理念与美学方向

### 2.1 设计理念
**"实用主义现代风"** - 融合工业建材美学与现代数字界面，强调清晰、高效、可靠的视觉体验。

### 2.2 美学方向
- **主题**：工业简约风 + 温暖商业氛围
- **特点**：
  - 清晰的信息层次结构
  - 大字体、高对比度，方便阅读
  - 工业风格的配色（深灰、蓝灰为主）
  - 温暖的橙色作为行动按钮和强调色
  - 简洁的图标和按钮，减少认知负担

## 6. 数据流设计

### 6.1 核心数据流程

```
进货流程:
供应商 → 进货单 → 库存增加 → 进货记录

销售流程:
选择商品 → 销售开单 → 库存减少 → 销售记录 → 生成利润

库存预警流程:
检查库存 → 低于阈值 → 预警提醒 → 补货/调整
```

### 6.2 状态管理

使用 Zustand 进行状态管理，主要store：

- **useProductStore** - 商品相关状态
- **useInventoryStore** - 库存相关状态
- **useSaleStore** - 销售相关状态
- **useCustomerStore** - 客户相关状态
- **useReportStore** - 报表相关状态
- **useAppStore** - 应用全局状态（侧边栏折叠、当前用户等）

### 17.1 客户价值评分算法
| 维度 | 权重 | 计算方式 |
|------|------|----------|
| 年度累计销售额 | 30% | 分数 = (当年销售额 / 目标销售额) × 30 |
| 平均利润率 | 25% | 分数 = (实际利润率 / 目标利润率) × 25 |
| 回款速度 | 20% | 分数 = (30 - 平均欠款天数) / 30 × 20 |
| 欠款逾期率 | 15% | 分数 = (1 - 逾期率) × 15 |
| 返点占比 | 10% | 分数 = (1 - 返点占比) × 10 |

| 中等价值 | △ | 60-84 | 稳定客户群 |
| 普通价值 | ○ | 40-59 | 普通客户群 |
| 低价值 | × | 0-39 | 需关注客户群 |

## 25. 联系人-结账主体模型（v4.3 新增）

### 25.1 架构设计背景

#### 25.1.1 现有架构问题
原 `Customer` 表混合了"人"和"付款主体"两个概念：
- 手机号可能重复创建
- 付款主体和个人关系不清晰
- 账目挂在谁身上不清楚
- 无法支持"谁买东西 ≠ 谁付钱"的业务场景

#### 25.1.2 核心设计原则
1. **人是独立的**：一个人只录一次，用手机号唯一标识
2. **身份是临时的**：一个人可以同时拥有多个身份、参与多个项目
3. **买付分离**：谁买东西 ≠ 谁付钱，付款方是独立的实体
4. **账目清晰**：所有账目都挂在"付款主体"下，而不是挂在"采购人"下

### 25.2 新数据结构（8表设计）

#### 25.2.1 Contact（联系人/人员）
一个人只有一个记录，手机号唯一。

**contactType 枚举定义**：
| 值 | 名称 | 说明 |
|----|------|------|
| `customer` | 散客/业主 | 普通购买客户 |
| `plumber` | 水电工 | 有返点的介绍人 |
| `other` | 其他 | 其他人员 |

> 注意：公司/装修公司是 Entity（结账主体），不是 Contact。Contact 是人，Entity 是组织。

### 26.1 业务背景
建材行业销售涉及多个独立角色，需要完整记录各角色信息以便追溯和返点计算。

### 26.2 销售单角色定义

#### 旧架构 (Sale)
| 角色 | 说明 | 关联字段 |
|------|------|----------|
| 购货人(业主) | 实际购买材料的客户 | buyerCustomerId |
| 付款人 | 实际付款方（装修公司或业主） | payerCustomerId |
| 介绍人 | 引荐客户的水电工或其他人 | introducerCustomerId |
| 提货人 | 实际到店取货的人员 | pickerName, pickerPhone, pickerType |

#### 新架构 (SaleOrder)
| 角色 | 说明 | 关联字段 |
|------|------|----------|
| 购货人 | 购买东西的人（**Contact**） | buyerId |
| 付款人 | 实际付款的人（**Contact**） | payerId |
| 介绍人 | 水电工等介绍人（**Contact**） | introducerId |
| 提货人 | 提货人（**Contact** 或手动输入） | pickerId, pickerName, pickerPhone |
| **结账主体** | **账目挂在谁身上（Entity）** | **paymentEntityId** |
| 项目 | 关联的项目（BizProject） | projectId |
